# Product Images - Local Bundle Setup

## 📁 Folder Structure
```
assets/images/products/
├── finished/          # Finished goods images
├── traded/            # Traded goods images  
└── placeholder.png    # Default fallback image
```

## ✅ How It Works

### 1. **Image Selection & Save**
- User selects image from device (max 500KB)
- App automatically copies to `assets/images/products/finished/` or `traded/`
- Filename format: `{ProductName}_{timestamp}.{ext}`
- Path saved in database: `localImagePath` field

### 2. **Bundled with App**
- All images in `assets/images/products/` are bundled with APK/EXE
- No Firebase Storage needed
- Works 100% offline
- Images distributed with installer

### 3. **View Modes**
Products can be viewed in 5 different modes:
- **Large Icons** (180x180) - Best for visual browsing
- **Medium Icons** (140x140) - Balanced view
- **Small Icons** (100x100) - Compact grid
- **List View** - Image + details in rows
- **Detailed Table** - Full data table (current)

## 🔧 Setup Instructions

### For Development:
1. Add product images to respective folders
2. Images auto-copy when user selects via app
3. Rebuild app to include new images

### For Production Build:
1. All images in `assets/` folder are automatically bundled
2. APK/EXE size increases based on total image size
3. Recommended: Optimize images before adding (use TinyPNG)

## 📝 Image Guidelines
- **Format**: PNG or JPG
- **Size**: Max 500KB per image
- **Dimensions**: 500x500px recommended
- **Naming**: Auto-generated by app
- **Fallback**: `placeholder.png` shown if image missing

## 🎯 Where Images Show
1. **Product Catalog** - All view modes
2. **Inventory Screen** - List view
3. **Product Edit Form** - Preview section
4. **Sales/Invoice** - Product selection

## ⚠️ Important Notes
- Images are NOT synced to Firebase
- Each device has its own bundled images
- Update app to distribute new images
- Old images remain in bundle (manual cleanup needed)
